# research/experiments/

Per-experiment writeups: hypothesis · setup · result · verdict.

- **Holds:** one markdown per experiment / experiment-family (e.g. `e40c_followup_experiments.md`, `f1_phase3a_results.md`, `p3b_pyramid_results.md`, `rs_multietf_batch_results.md`).
- **Goes here:** the narrative behind a run — what was tried, why, what happened, accept/reject. Each pinned to (commit + data-fingerprint + bt-id) so it is traceable.
- **Does NOT:** the catalog index (→ `../catalog/`), parity diagnostics (→ `../parity/`), trade-behavior studies (→ `../trade-analysis/`), or the numeric ledger (→ `results/`).
